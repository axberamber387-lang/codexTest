# Спецификация проекта `codexTest`

## 1. Назначение проекта

`codexTest` — это браузерная игра Snake, реализованная на чистом JavaScript без сторонних UI-фреймворков. Проект включает:

- клиентскую часть с отрисовкой поля, управлением, сменой темы и звуком;
- модуль игровой логики с чистыми функциями;
- минималистичный Node.js сервер для локальной раздачи статических файлов;
- набор автотестов для ключевых правил движения змейки.

## 2. Цели и ожидаемый результат

Проект должен позволять пользователю:

- открыть игру в браузере через локальный HTTP-сервер;
- запустить новую сессию;
- управлять змейкой с клавиатуры и экранных кнопок;
- переключать скорость игры;
- включать и выключать звук;
- менять цветовую тему интерфейса;
- видеть текущий счет и статус игры;
- перезапускать игру после завершения или в любой момент.

## 3. Технологический стек

- `Node.js` с ESM-модулями
- нативный `node:http` для локального сервера
- HTML + CSS + Vanilla JavaScript
- Web Audio API для генерации звуков и простой фоновой мелодии
- встроенный `node:test` для тестирования

## 5. Архитектура

### 5.1. Клиентский слой

Файл `src/main.js` отвечает за:

- хранение текущего состояния интерфейса и игры;
- запуск и остановку игрового цикла через `setInterval`;
- обработку клавиатуры и экранных кнопок;
- управление скоростью, темой и звуком;
- отрисовку поля и служебных статусов;
- воспроизведение звуковых эффектов и фоновой мелодии.

### 5.2. Игровая логика

Файл `src/snakeLogic.js` содержит чистые функции, которые не зависят от DOM:

- создание начального состояния;
- нормализация направления движения;
- расчет следующего шага;
- перенос координат по краям поля;
- проверку столкновений;
- выбор новой позиции для еды.

### 5.3. Серверный слой

Файл `server.js` поднимает локальный HTTP-сервер и раздает статические файлы проекта из корневой директории. Сервер:

- отдает `index.html` по пути `/`;
- определяет MIME-тип по расширению;
- возвращает `404`, если файл не найден или путь указывает на директорию.

## 6. Функциональные требования

### 6.1. Игровое поле

- Размер сетки: `16 x 16`.
- Поле должно перерисовываться целиком на каждом игровом тике.
- Голова змейки должна визуально отличаться от тела.
- Еда должна отображаться отдельным стилем.

### 6.2. Начальное состояние

При первичной загрузке:

- игра находится в статусе `idle`;
- счет равен `0`;
- на экране отображается overlay с кнопкой `Start game`;
- направление по умолчанию — `right`;
- начальная длина змейки — `3` сегмента.

### 6.3. Запуск и перезапуск

- Кнопка `Start game` запускает новую игровую сессию и переводит статус в `running`.
- Кнопка `Restart` сбрасывает игру в начальное состояние `idle`.
- При рестарте активные таймеры игры и музыки должны останавливаться.

### 6.4. Управление

Поддерживаются два способа управления:

- клавиши `ArrowUp`, `ArrowDown`, `ArrowLeft`, `ArrowRight`;
- клавиши `W`, `A`, `S`, `D`;
- экранные кнопки `Up`, `Down`, `Left`, `Right`.

Дополнительно:

- `Space` переключает паузу, если игра уже запущена;
- прямой разворот в противоположную сторону за один ход запрещен.

### 6.5. Движение и столкновения

- Змейка двигается с фиксированным интервалом, зависящим от выбранной скорости.
- Поддерживаются скорости:
  - `slow` — `220 ms`;
  - `normal` — `140 ms`;
  - `fast` — `90 ms`.
- При выходе за границу поля змейка появляется с противоположной стороны.
- При столкновении головы с телом игра переходит в статус `game-over`.
- Переход в бывшую клетку хвоста разрешен, если на этом ходе еда не съедается.

### 6.6. Еда и счет

- Если голова змейки достигает клетки с едой, длина змейки увеличивается на `1`.
- При съедании еды счет увеличивается на `1`.
- Новая еда должна появляться только в свободной клетке.
- Если свободных клеток не осталось, функция размещения еды возвращает `null`.

### 6.7. Статусы игры

Поддерживаются следующие статусы:

- `idle` — ожидание запуска;
- `running` — активная игра;
- `paused` — игра поставлена на паузу;
- `game-over` — проигрыш.

В интерфейсе должны отображаться текстовые состояния:

- `Press Start game`
- `Running`
- `Paused`
- `Game over`

### 6.8. Звук

- Звук по умолчанию включен.
- Пользователь может переключать состояние `On/Off`.
- При включенном звуке должны проигрываться:
  - короткий эффект при съедании еды;
  - сигнал завершения игры;
  - циклическая фоновая мелодия во время активной игры.
- При выключении звука музыкальный цикл должен останавливаться.

### 6.9. Темы оформления

Поддерживаются темы:

- `default`
- `eggplant`
- `barbie`
- `jaundice`

Смена темы должна изменять CSS-переменные страницы и применяться без перезагрузки.

## 7. Пользовательский сценарий

1. Пользователь запускает сервер командой `npm run dev`.
2. Открывает игру в браузере по адресу `http://localhost:3000`.
3. Выбирает скорость, звук и тему.
4. Нажимает `Start game`.
5. Управляет змейкой клавишами или экранными кнопками.
6. Набирает очки, собирая еду.
7. При необходимости ставит игру на паузу клавишей `Space`.
8. После проигрыша или по желанию нажимает `Restart`.

## 8. Нефункциональные требования

- Проект должен запускаться без сборщика и без дополнительных зависимостей.
- Игровая логика должна быть отделена от DOM и пригодна для unit-тестов.
- Интерфейс должен быть адаптивным для узких экранов.
- Код должен использовать ES-модули.

## 9. Команды запуска

```bash
npm install
npm run dev
```

Для запуска тестов:

```bash
npm test
```

## 10. Тестовое покрытие

Автотесты должны подтверждать:

- запрет прямого разворота;
- движение в выбранном направлении;
- увеличение длины и счета после поедания еды;
- корректный переход через границы поля;
- размещение еды только в свободной клетке;
- корректную работу вспомогательных функций;
- допустимость перехода в клетку хвоста при обычном ходе.

## 11. Ограничения текущей реализации

- Данные игры не сохраняются между перезагрузками страницы.
- Таблица рекордов отсутствует.
- Нет экрана победы при полном заполнении поля змейкой.
- Статический сервер не реализует кеширование, логирование запросов и production-настройки.
- Overlay показывается только в состоянии `idle`; при `game-over` пользователь ориентируется по статусу и кнопке `Restart`.

## 12. Критерии приемки

Проект считается соответствующим спецификации, если:

- сервер поднимается локально без ошибок;
- в браузере отображается игровое поле и элементы управления;
- игра запускается, ставится на паузу и перезапускается;
- счет увеличивается при поедании еды;
- столкновение с телом завершает игру;
- переключение скорости, звука и темы работает во время сессии;
- автотесты `npm test` проходят успешно.
